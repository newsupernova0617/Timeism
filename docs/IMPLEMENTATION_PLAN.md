## 3. 애드센스 승인 전략

### 3.1 핵심 문제점 & 해결책

> **문제**: 구글 봇은 '기능(시계)'을 콘텐츠로 인정하지 않음 (Low Value Content).  
> **해결**: 사용자에게 방해되지 않는 **하단 영역**에 고품질 텍스트 콘텐츠를 배치.

#### 전략 1: 메인 페이지 텍스트 보강 (UI Integrated)
- **위치**: Fold(첫 화면) 아래쪽 영역
- **방식**: 단순 나열이 아닌 **카드 UI** 또는 **아코디언 UI** 활용
- **내용**: 서버 시간 정의, 원리, 티켓팅 꿀팁 (1,500자 이상)

#### 전략 2: 타겟별 전용 페이지 생성 (SEO)
- `/ko/interpark`, `/en/ticketmaster` 등 다국어 지원 구조와 결합
- 각 사이트별 맞춤형 공략법 제공

## 4. 기술적 고도화 계획 (New)

### 4.1 다국어 지원 (i18n & SEO)

글로벌 트래픽 확보와 애드센스 승인을 동시에 노립니다.

1.  **URL 구조 변경**:
    *   기본: `timeism.com` (브라우저 언어 감지 후 리다이렉트 또는 영어)
    *   한국어: `timeism.com/ko`
    *   영어: `timeism.com/en`
    
2.  **`hreflang` 태그 적용**:
    *   구글이 언어별 페이지를 정확히 인식하도록 헤더에 추가
    ```html
    <link rel="alternate" hreflang="en" href="https://timeism.com/en/" />
    <link rel="alternate" hreflang="ko" href="https://timeism.com/ko/" />
    <link rel="alternate" hreflang="x-default" href="https://timeism.com/" />
    ```

3.  **`lang` 속성 동적 변경**:
    *   `<html>` 태그의 `lang` 속성을 URL에 맞춰 `ko`, `en` 등으로 올바르게 설정

### 4.2 성능 최적화 (Core Web Vitals)

애드센스와 검색 상위 노출을 위해 성능 점수 관리 필수.

1.  **Lighthouse 점수 관리**:
    *   Performance, Accessibility, SEO 모두 95점 이상 유지
2.  **이미지 최적화**:
    *   WebP 포맷 사용
    *   적절한 `width`, `height` 명시로 CLS(Layout Shift) 방지
3.  **리소스 최적화**:
    *   CSS/JS Minification (이미 적용됨)
    *   불필요한 폰트 로딩 방지 (시스템 폰트 우선 또는 `font-display: swap`)

## 5. UI/UX 통합 전략 (New)

> **핵심 원칙**: "Rich Aesthetics"를 유지하며 콘텐츠를 추가한다. 텍스트 폭탄처럼 보이지 않게 디자인한다.

### 5.1 메인 콘텐츠 영역 (SEO Text)

단순한 `<p>` 태그의 나열은 사이트의 프리미엄 느낌을 해칩니다. 다음과 같은 UI 패턴을 사용합니다.

*   **Feature Grid (카드형)**:
    *   "정밀한 시간", "NTP 동기화", "알람 기능" 등의 주제를 아이콘과 함께 카드 형태로 배치.
    *   마우스 호버 시 미세한 인터랙션(Scale up, Border highlight) 적용.
*   **Accordion (Faq 섹션)**:
    *   긴 텍스트 설명은 기본적으로 접어두고, 제목을 클릭하면 부드럽게 펼쳐지도록 구현.
    *   공간 절약과 정보 전달을 동시에 잡음.
*   **Typography Hierarchy**:
    *   섹션 제목, 소제목, 본문의 폰트 크기와 색상 대비(Contrast)를 명확히 하여 가독성 증대.
    *   `tokens.css`의 컬러 팔레트(`--text-primary`, `--text-secondary`)를 철저히 준수.

### 5.2 타겟별 전용 페이지 템플릿

모든 타겟 페이지(`interpark`, `yes24` 등)는 일관된 고품질 레이아웃을 공유합니다.

*   **Hero Section**: 사이트 로고 + 실시간 시계 (가장 강조)
*   **Sticky Info Bar**: "0.1초 차이가 승패를 가릅니다" 같은 중요 팁을 스크롤 시 상단 고정.
*   **Glassmorphism**: 콘텐츠 컨테이너에 반투명 배경 효과를 주어 배경(Gradient/Blur)과 어우러지게 함.

### 5.3 모바일 최적화

*   긴 텍스트는 모바일에서 스크롤 압박이 됨.
*   **"더 보기" 버튼**을 활용하여 초기 로딩 시에는 핵심 요약만 보여주고 사용자가 원할 때 전체 글을 로드.

## 6. Phase별 구현 계획

### Phase 1: 기반 구조 및 SEO (Week 1)
1.  **다국어 라우팅 시스템 구축** (`/ko`, `/en`)
    *   Express 라우터 개편
    *   `i18n` JSON 파일 구조 생성
2.  **SEO 태그 시스템 고도화**
    *   `hreflang`, `canonical`, `lang` 속성 동적 생성 로직 추가
3.  **메인 템플릿 개편 (콘텐츠 섹션 추가)**
    *   기존 디자인을 유지하며 하단에 그리드/아코디언 섹션 추가 디자인 및 코딩

### Phase 2: 콘텐츠 확충 (Week 2)
4.  **타겟별 전용 페이지 6종 생성**
    *   인터파크, 멜론, 예스24 등 (한국어/영어 콘텐츠 작성)
5.  **블로그/가이드 페이지 시스템**
    *   MD 파일 또는 EJS 기반의 가벼운 블로그 시스템
    *   티켓팅 가이드 글 3~5편 작성

### Phase 3: 성능 및 디테일 (Week 3)
6.  **성능 최적화**
    *   Lighthouse 측정 및 개선
    *   이미지 포맷 변환 및 레이지 로딩
7.  **UX 디테일 잡기**
    *   호버 이펙트, 트랜지션 점검
    *   다크 모드 대응 확인
